# Title: AppFooter rail-aware visibility and right-aligned paimos.com logo link

## Description

Two refinements to `AppFooter.vue` and its placement on the project detail page:

1. **Hide AppFooter when the workspace rail is shown.** `ProjectDetailView.vue`
   currently renders `<AppFooter compact />` directly above the
   `pd-workspace-rail` button strip. Showing both stacks two
   logo/branding strips on top of each other and makes the rail feel
   crowded. The footer should not be rendered at all whenever the rail
   is rendered (which, on the project detail view, is always).

2. **Right-align the footer logo and link it to paimos.com.** The
   logo currently sits to the left of the company name. Move it to the
   right edge of the footer, wrap it in an anchor pointing to
   `https://paimos.com` (target=_blank, rel=noopener), and keep the
   company name text on the left. The non-compact and compact
   variants should both adopt the new layout.

## Priority

P40

## Acceptance Criteria

- [ ] On `ProjectDetailView`, `AppFooter` is never rendered while
      `pd-workspace-rail` is on screen.
- [ ] On every other view that renders `<AppFooter />`, the layout
      shows the company name on the left and the logo on the right
      (via `justify-content: space-between` or equivalent).
- [ ] The logo is wrapped in an anchor with `href="https://paimos.com"`,
      `target="_blank"`, and `rel="noopener"` (no `noreferrer` so
      paimos.com can attribute referral traffic from the app).
- [ ] Hover/focus state on the link is visually distinct (cursor
      pointer, subtle opacity bump) so users discover the link.
- [ ] No regressions in `npm run typecheck`, `npm test`, or
      `npm run build`.

## Additional Requirements

- Touches `frontend/src/components/AppFooter.vue` and
  `frontend/src/views/ProjectDetailView.vue` only.
- Compact variant inside `pd-workspace-footer` will be removed in
  full as part of (1), so the compact-specific CSS in `AppFooter.vue`
  can stay in place for other future consumers but no longer needs
  to be wired up on ProjectDetailView.
