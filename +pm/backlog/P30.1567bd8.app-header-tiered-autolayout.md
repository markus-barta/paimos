# Title: AppHeader tiered autolayout — height-stable, container-aware collapse

## Description

`AppHeader` doesn't degrade gracefully when its container shrinks (the
canonical case being the issue side panel pinned, which now shrinks
`.main` after `+pm/P30.c6aad81`). Two things break:

1. **Right cluster wraps.** `.ah-right` uses `flex-wrap: wrap`, so when
   the per-view teleport content (customer pill, "updated Xh ago",
   issue-key chip, status badge, overflow menu) plus the global Undo
   button can't fit on one line, Undo drops to a second row. The
   header grows from 52px to ~85px.

2. **Layout reads viewport, not container.** The grid template uses
   `minmax(260px, 32vw)` for the center column and the only
   responsive breakpoint is a viewport `@media (max-width: 900px)`.
   Pinning the panel shrinks `.main` but leaves the viewport at full
   width, so neither the `32vw` floor nor the media query react.

The replacement is **container-query driven**, with a fixed header
height and a four-tier hierarchy of sacrifice — decoration drops
before content, content shortens before disappearing.

## Priority

P30

## Acceptance Criteria

- [ ] `AppHeader` height is locked at 52px on desktop; right cluster
      never wraps to a second row, regardless of pin state.
- [ ] Tiered collapse driven by `@container` queries on `.main`:
  - **Tier 1** (`< 1100px`): `.ah-subtitle` + `TagChip`s hide; search
    shrinks 280→220 (focused 380→300).
  - **Tier 2** (`< 920px`): `.ah-title` ellipsis-truncates at ~14ch;
    "updated Xh ago" prefix drops, the issue-key link chip stays;
    customer pill becomes icon-only (link preserved).
  - **Tier 3** (`< 760px`): search collapses to a 36px icon-button
    that expands inline on focus; Undo button drops the "Undo" label,
    keeps icon + count.
  - **Tier 4** (`< 600px`): `.ah-title` hides; the project key badge
    alone identifies the view.
- [ ] Container queries are gated to desktop layout — the existing
      `@media (max-width: 900px)` mobile stack (multi-row,
      full-width search) is unchanged.
- [ ] A subtle right-edge mask on `.ah-left` softens breadcrumb
      truncation (Linear-style fade rather than hard ellipsis cut).
- [ ] `.ah-search-wrap` width and the various `display`/`opacity`
      transitions are eased (≈180ms) so tier crossings don't pop.
- [ ] No regressions in `npm run typecheck`, `npm test`, or
      `npm run build`.
