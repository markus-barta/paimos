# Title: AppHeader does not shrink when issue side panel is pinned

## Description

When the issue side panel (`IssueSidePanel`, the `.sp-content` slide-in
on the right) is **pinned**, only the issue list content reflows out of
the way — the top `AppHeader` (back arrow, project breadcrumb, search
box) stays full-width and gets visually covered by the panel.

Root cause: `AppHeader` is a sibling of `.main-content` inside `.main`
(see `AppLayout.vue:202-212`). The pinned panel is `position: fixed` so
it never pushes layout. The current shrink-when-pinned logic lives on
`.issue-list-root` only (`IssueList.vue:806`, `marginRight` inline
style), so it shrinks the list but not the header.

Fix is to lift the pinned-panel offset one level up: apply the offset
as `padding-right` on `.main`, so both `AppHeader` and `.main-content`
shrink together. This requires the pinned-state to be reachable from
`AppLayout`, which today only lives inside `IssueList`.

## Acceptance Criteria

- [ ] When the issue side panel is pinned and an issue is open, the
      `AppHeader` visibly shrinks so its right edge aligns with the
      left edge of the pinned `.sp-content` (no overlap).
- [ ] The same offset applies to `.main-content`, so the existing
      issue list layout still works (no double-shrink, no gap).
- [ ] Resizing the pinned panel (drag handle) re-flows the header
      width live with the same width that drives the list offset.
- [ ] Unpinning the panel restores the header to full width.
- [ ] Closing the panel (x button or navigating away) restores the
      header to full width even if pinned was true.
- [ ] No regressions in `npm run typecheck`, `npm test`, or
      `npm run build`.

## Additional Requirements

- New composable `useSidePanelPinned` (singleton) parallel to
  `useSidePanelWidth`, exposing `pinned` + `visible` refs and a setter
  that mirrors `LS_SIDEBAR_PINNED`. `IssueList.vue` reads/writes via
  the composable; `AppLayout.vue` reads it and applies inline
  `padding-right` to `.main`.
- Drop the now-redundant `marginRight` style on `.issue-list-root`.
- Touches:
  - `frontend/src/composables/useSidePanelPinned.ts` (new)
  - `frontend/src/components/AppLayout.vue`
  - `frontend/src/components/IssueList.vue`
